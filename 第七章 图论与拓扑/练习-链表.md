# 练习 1 — 链表作为一种拓扑特例

Exercise 1 — The Linked List as a Topological Special Case

教材通常将链表介绍为一种"序列容器"。但链表首先是一种**图**——一种边数被高度约束的特殊图。

Textbooks typically introduce the linked list as a "sequential container." But a linked list is first and foremost a **graph** — a highly constrained special case of a graph.

以下问题中，"链表"指的是单向链表（每个节点有一个指向后继的指针），"节点数"= N。

In the following, "linked list" means a singly linked list (each node has one pointer to its successor), and "node count" = N.

---

**(1)** 单向链表是本章定义的哪种图？（树？DAG？全序？）

A singly linked list is which kind of graph as defined in this chapter? (Tree? DAG? Total order?)

*提示：分支因子。Hint: branching factor.*

---

**(2)** 双向链表（每个节点有前驱和后继指针）是什么图？

What kind of graph is a doubly linked list (each node has prev and next pointers)?

*提示：环？树？Hint: cycle? tree?*

---

**(3)** 环形链表的 N 个节点有几条边？它还是树吗？

How many edges does a circular linked list with N nodes have? Is it still a tree?

*提示：树的边数 = N−1。Hint: a tree has N−1 edges.*

---

**(4)** 从图论的角度，数组和单向链表的本质区别是什么？（设想数组的随机访问是一次"边"，指向任意位置的跳转）

From a graph-theoretic perspective, what is the essential difference between an array and a singly linked list? (Treat random access in an array as an "edge" — a jump to any position.)

*提示：每个节点有多少条"出边"？Hint: how many outgoing "edges" does each node have?*

---

**(5)** 添加额外的"边"会增加遍历灵活性，也会增加约束的复杂度。你能给出一个例子吗？

Adding extra "edges" increases traversal flexibility, but also increases constraint complexity. Can you give an example?

*提示：寻找链表、二叉树、DAG之间的状态变化。Hint: think about the state transitions among linked list, binary tree, and DAG.*
