# 语言与抽象

Language and Abstraction

语言的作用：通过事先约定共识，来缩减达成共识所需要的时间成本。想象一下，如果没有语言， 人和人沟通交流是多么的麻烦！

Language's function: through pre-established conventions, it reduces the time cost of reaching consensus. Imagine, without language, how cumbersome communication between people would be!

---

有时候，小圈子内形成了一整套”黑话”，虽然提高了小圈子里的沟通效率，却让圈子内外的沟通成本变得极高。

Sometimes, an entire set of “jargon” forms within a small circle. While it raises communication efficiency inside the circle, it makes communication between insiders and outsiders extremely costly.

语言往往通过一个简洁的概念压缩了大量相似的事物，因此具备模糊性。模糊性是效率的障碍，但同时也是联想、创造和幽默的来源。有些时候，语言的模糊性也是社会能妥协运转下去的前提。

Language often compresses a large number of similar things into one concise concept, and thus inherently possesses ambiguity. Ambiguity is an obstacle to efficiency, but it is also a source of association, creativity, and humor. Sometimes, the ambiguity of language is also the precondition for society to compromise and function.

---

医学领域里的沟通，追求极致的精确。外科手术里，切错一个器官，就是医疗事故。

Communication in medicine pursues extreme precision. In surgery, cutting the wrong organ is a medical error.

而电影、戏剧、诗歌里的词句，则往往追求含蓄、隐喻，令人落泪或微笑。

The words in film, theater, and poetry, by contrast, often pursue subtlety and metaphor, moving people to tears or smiles.

---

传统的计算机行业里，要求从业者掌握一套精确的计算机编程语言作为基本功。但计算机科学与技术发展的历史，就是对大多数从业者的语言精确程度不断下降的历史。

In the traditional computing industry, practitioners were required to master a precise computer programming language as a basic skill. Yet the history of computer science and technology is a history of continuously lowering the required precision of language for most practitioners.

一代接一代的程序员，不断在更高的抽象层次上工作，而较低的抽象层次不断被自动化。

Generation after generation of programmers have worked at progressively higher levels of abstraction, while the lower levels have been continuously automated.

---

最早的程序员需要直接编制程序指令里的每个0和1。

The earliest programmers needed to directly craft each 0 and 1 in the program instructions.

然后，人们首先发明了汇编器，程序员从手动输入每个0和1，变成只需要输入一些简短的指令。

Then, the assembler was invented. Programmers went from manually entering every 0 and 1 to only needing to type a few short instructions.

---

随后，又发明了抽象层次更高的编程语言，如Fortran，C语言等等，通过编译器的转换，程序员写下一句程序，可以自动转换成几十条对应的汇编语句。

Subsequently, higher-level programming languages were invented, such as Fortran, C, and so on. Through the compiler's transformation, a single line written by the programmer could be automatically converted into dozens of corresponding assembly instructions.

今天，借助大语言模型，一句准确的自然语言描述，可以被很快地转换为几十行甚至几百行C语言或Python语言的代码，实现较为复杂的功能。

Today, with the help of large language models, one precise natural language description can be quickly converted into tens or even hundreds of lines of C or Python code, implementing fairly complex functionality.

我们强调了”准确的”自然语言描述。这就是说，并不是所有自然语言都能够被转换成好的程序。只有一段自然语言，合理地描述了一种可以被实现的功能需求时，人工智能工具才能将它转换成满足人类需求的代码。

We emphasize “precise” natural language description. That is, not all natural language can be converted into good programs. Only when a piece of natural language reasonably describes a functional requirement that can be implemented, can the AI tool convert it into code that meets human needs.

---

我们注意到，编译器其实将编程语言设计者的品味施加给了所有的普通程序员——编译器相当于一名严厉的教师，要求你写出来的代码，必须符合编程语言设计者的品味。这使得，尽管大多数的普通程序员都不具备写出优秀代码的能力，但软件的质量依然能够得到保证。

We note that the compiler, in effect, imposes the programming language designer's taste upon all ordinary programmers — the compiler acts as a strict teacher, demanding that the code you write must conform to the language designer's taste. This ensures that, even though most ordinary programmers lack the ability to write excellent code, software quality can still be guaranteed.

---

一个基本事实是，有些人使用AI编程会写出糟糕的代码，有些人会写出优秀的代码。因此，有些自然语言的用法，能让AI生成优秀的代码，另一些会让AI生成糟糕的代码。

A basic fact: some people produce terrible code when programming with AI, while others produce excellent code. Therefore, certain ways of using natural language can lead AI to generate excellent code, while others lead AI to generate terrible code.

---

如果”好的自然语言用法能够被总结成一些规则”，我们就发明了一种新的”用于AI编程的语言”。或者说，这是一种”有限制的自然语言”。

If “good natural language usage patterns can be summarized into a set of rules,” we have invented a new kind of “language for AI programming.” Or rather, this is a “restricted natural language.”
