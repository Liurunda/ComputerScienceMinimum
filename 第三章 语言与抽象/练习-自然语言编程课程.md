# 练习 5 — 一门"不出现代码"的程序设计入门课是否可能？

Exercise 5 — Is an Introductory Programming Course "Without Visible Code" Possible?

1950 年代，汇编程序员的主流反应是：编译器生成的代码不可能比手工汇编快。1970 年代，结构化编程的反对者认为 GOTO 是程序员不可剥夺的自由。1990 年代，C 程序员批评 Java 程序员不懂内存管理。今天，传统程序员批评 LLM 用户不懂代码。

In the 1950s, the mainstream reaction of assembly programmers was: compiler-generated code could never match hand-written assembly. In the 1970s, opponents of structured programming argued that GOTO was a programmer's inalienable freedom. In the 1990s, C programmers criticized Java programmers for not understanding memory management. Today, traditional programmers criticize LLM users for not understanding code.

每一代抽象的出现，都伴随着上一代从业者对"失去底层控制"的焦虑。每一代最终也都接受了：当新的抽象层在绝大多数场景中足够好时，"底层只有极端情况才需要"就变成了共识。

Every new layer of abstraction is accompanied by anxiety from the previous generation about "losing low-level control." And every generation eventually accepted: when the new abstraction layer is good enough in the vast majority of scenarios, "the low level is only needed for extreme cases" becomes the new consensus.
