# 练习 3 — 设计一个"易学难精"的交互

Exercise 3 — Design an "Easy to Learn, Hard to Master" Interaction

本章的四项原则——状态可见、反馈及时、复杂度可学、代理感不可剥夺——适用于设计任何交互系统。

This chapter's four principles — visible state, timely feedback, learnable complexity, unalienable agency — apply to designing any interactive system.

---

**(1)** 选一个你日常使用但觉得"不够好"的工具或应用（可以是代码编辑器、聊天软件、文件管理器等）。用本章的四条原则，诊断它最需要改进的一点。

Pick a tool or app you use daily but find "not good enough" (e.g., a code editor, chat software, file manager). Use this chapter's four principles to diagnose the one thing most in need of improvement.

---

**(2)** 为这个工具设计一个"新手模式"——一个只暴露最核心功能的简化界面，让新手在 5 分钟内完成第一个有用任务。列出你删掉了哪些功能，保留了哪些，为什么。

Design a "novice mode" for this tool — a simplified interface that exposes only the core functionality, enabling a novice to complete their first useful task within 5 minutes. List what features you removed, what you kept, and why.

---

**(3)** 同一个工具，它的"专家模式"是随着掌握深入而自然浮现出的"高级技巧"——不需要单独一个按钮切换为"专家版"。你的设计中，专家模式的"深度"从哪儿来？新手如何自然地发现它们？

For the same tool, describe the "advanced techniques" that naturally emerge with mastery — no need for a separate "expert mode" button. Where does the "depth" of expertise come from in your design? How would novices naturally discover them?

---

**(4)** （可选：实现）如果你在练习 1 中选择了代码编辑器或类似工具，用第三章的 CNL 设计原则为一款"支持代码编辑的 AI 助手"设计一个交互原型——写出示例 prompt 序列，展示用户如何在一轮对话中体会到状态可见、反馈及时、复杂度可学、代理感。

(Optional: implement) If you chose a code editor or similar tool in Exercise 1, use the CNL design principles from Chapter 3 to design an interaction prototype — write a sample prompt sequence showing how a user can experience visible state, timely feedback, learnable complexity, and agency within a single dialogue session.
