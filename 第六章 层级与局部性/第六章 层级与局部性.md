# 第六章 层级与局部性

Chapter 6: Hierarchy and Locality

一个程序执行时产生的内存地址序列，不是随机的。

The sequence of memory addresses produced by an executing program is not random.

这是一个可观测的事实，不是一个哲学立场。如果你记录下任何一个真实程序——编译器、数据库、游戏——在一微秒间隔内引用的内存地址，你会发现：**下一个地址大概率接近上一个地址。** 这不是概率上的假设，这是过去六十年所有程序行为测量一致给出的结果。

This is an observable fact, not a philosophical stance. If you record the memory addresses referenced by any real program — a compiler, a database, a game — at microsecond intervals, you will find: **the next address is likely near the previous one.** This is not a probabilistic assumption; it is the consistent result of measuring program behavior over the past six decades.

用信息论的语言：给定程序在时刻 t 访问的地址 addr_t，它在时刻 t+1 访问的地址 addr_{t+1} 的条件熵 H(addr_{t+1} | addr_t)，远小于无条件熵 H(addr)。局部性不是一种"直觉"——它是地址引用序列中可测量的条件概率密度的集中。

In the language of information theory: given the address addr_t accessed by a program at time t, the conditional entropy H(addr_{t+1} | addr_t) of the address accessed at t+1 is far smaller than the unconditional entropy H(addr). Locality is not an "intuition" — it is the concentration of conditional probability density in the address reference stream, measurable and reproducible.

---

有两种不同的局部性，分别对应两个不同的统计模式。

There are two distinct kinds of locality, corresponding to two different statistical patterns.

**时间局部性**：如果一个地址被访问了，它在不久的将来很可能再次被访问。H(addr_{t+k} | addr_t) 在 k 较小时显著低于无条件熵。

**Temporal locality**: if an address is accessed, it is likely to be accessed again in the near future. H(addr_{t+k} | addr_t) is significantly lower than unconditional entropy for small k.

**空间局部性**：如果一个地址被访问了，它附近的地址在不久的将来很可能被访问。H(addr_{t+1} | addr_t) 在 addr_{t+1} 接近 addr_t 时显著低于在 addr_{t+1} 远离 addr_t 时。

**Spatial locality**: if an address is accessed, nearby addresses are likely to be accessed soon. H(addr_{t+1} | addr_t) is significantly lower when addr_{t+1} is near addr_t than when it is far.

这两个性质是可独立测量的。有些程序（如顺序扫描大数组）有很强的空间局部性但几乎没有时间局部性。有些程序（如在一个小哈希表上反复查询）有很强的时间局部性但空间局部性弱。

These two properties are independently measurable. Some programs (e.g., sequentially scanning a large array) have strong spatial locality but almost no temporal locality. Some programs (e.g., repeatedly querying a small hash table) have strong temporal locality but weak spatial locality.

---

局部性使得**层级**成立。

Locality makes **hierarchy** work.

如果程序访问内存的模式是随机的——如果 H(addr_{t+1} | addr_t) = H(addr) ——那么任何缓存都无效。你需要的每个数据都必须从最底层（最慢、最大容量）的存储中获取。这正是第二章所说的：随机 = 不可压缩。一个没有局部性的程序，其内存访问模式不具有可压缩的结构。

If a program's memory access pattern were random — if H(addr_{t+1} | addr_t) = H(addr) — then no cache would work. Every piece of data needed would have to be fetched from the lowest (slowest, largest) level of storage. This is precisely what Chapter 2 established: random = incompressible. A program without locality has a memory access pattern with no compressible structure.

真实程序有局部性。因此你可以建造一个层级：将频繁访问的数据放在靠近处理器的、快速但容量小的存储中；将不常访问的数据放在远离处理器的、慢速但容量大的存储中。你不需要为"哪个地址会出现在顶层"而猜测——你可以观测地址引用的历史，并且预测即将到来的引用。

Real programs have locality. Therefore you can build a hierarchy: place frequently accessed data in storage that is close to the processor, fast, and small; place infrequently accessed data in storage that is farther, slower, and larger. You do not need to guess which addresses will appear at the top — you can observe the history of address references and predict upcoming ones.

---

一个现代的存储层级如下。数字是数量级，精确值在具体硬件上略有不同，但比例关系是稳定的。

A modern memory hierarchy is as follows. Numbers are orders of magnitude; exact values vary slightly across hardware, but the ratios are stable.

| 层级 Level | 容量 Capacity | 延迟 Latency | 每层跳跃的延迟倍率 Latency ratio per step |
|---|---|---|---|
| 寄存器 Register | ~100 B | ~0.3 ns | — |
| L1 缓存 L1 Cache | ~64 KB | ~1 ns | ~3× |
| L2 缓存 L2 Cache | ~256 KB | ~4 ns | ~4× |
| L3 缓存 L3 Cache | ~16 MB | ~12 ns | ~3× |
| 主存 DRAM | ~16 GB | ~100 ns | ~8× |
| SSD | ~1 TB | ~100 μs | ~1,000× |
| 磁盘/磁带 Disk/Tape | ~10 TB | ~10 ms | ~100× |

**每下一层，容量增加约三个数量级，访问延迟增加约一个数量级。** 这个比例不是任意市场选择的结果——它最终受限于物理：越大的存储器，必然在物理上越远（光速上限），或必然需要越多的解码逻辑（电路复杂度上限）。

**Each step down increases capacity by approximately three orders of magnitude, and access latency by approximately one order of magnitude.** These ratios are not arbitrary market choices — they are ultimately constrained by physics: larger memory must be physically farther away (speed-of-light bound), or must require more decoding logic (circuit complexity bound).

这和第五章结尾的尺度表同构。区别在于：第五章的"芯片→分布式→互联网"是按**地理尺度**组织的层级；这一章的表是按**访问延迟**组织的层级。而两者的共同前提都是一样的：系统能够依赖可预测的局部性来将频繁操作分离到更快的层级。

This is isomorphic to the scale table at the end of Chapter 5. The difference: Chapter 5's "chip → distributed → internet" is a hierarchy organized by **geographic scale**; this chapter's table is a hierarchy organized by **access latency**. The common premise is identical: the system can rely on predictable locality to segregate frequent operations into faster levels.

---

层级带来了一个后果：**缓存一致性**。

Hierarchy brings a consequence: **cache coherence**.

当一个 CPU 有多个核——每个核拥有自己私有的 L1 缓存——就会产生一个多副本问题。核 A 写入地址 X，更新了它本地的 L1 副本。核 B 的 L1 中持有 X 的旧值。此时，哪一个副本是"正确的"？

When a CPU has multiple cores — each with its own private L1 cache — a multi-copy problem arises. Core A writes to address X, updating its local L1 copy. Core B's L1 holds the old value of X. At this moment, which copy is "correct"?

MESI 协议（及其变体）用以下方式回答：在被写入之前，缓存行处于 E（Exclusive，独占且干净）或 S（Shared，共享）状态。写入动作将状态转为 M（Modified，已修改），并通过总线消息使所有其他缓存的该缓存行失效（转为 I，Invalid）。此后，任何核试图读取 X 时，必须先从持有 M 副本的核获取最新值。

The MESI protocol (and its variants) answers this as follows: before being written, a cache line is in E (Exclusive, clean) or S (Shared) state. A write transitions the state to M (Modified) and sends bus messages to invalidate (transition to I, Invalid) that cache line in all other caches. Thereafter, any core attempting to read X must first obtain the latest value from the core holding the M copy.

这和第五章的 Lamport 逻辑时钟有一个关键的结构差异。Lamport 的协议被设计为在**不受控的延迟**下工作——它接受了偏序，因为全序在分布式系统中的代价不可承受。MESI 在**受控的延迟**下工作——核间总线的仲裁延迟是一个已知的、有上界的工程量（纳秒级）。它拒绝偏序，用总线锁定强行维持全序的幻象，并付出功耗作为代价。

There is a key structural difference from Lamport's logical clocks in Chapter 5. Lamport's protocol was designed to operate under **uncontrolled delay** — it accepted partial order because total order is prohibitively expensive in a distributed system. MESI operates under **controlled delay** — the arbitration latency of the inter-core bus is a known, bounded engineering quantity (nanoseconds). It rejects partial order, forcibly maintaining the illusion of total order through bus locking, and pays the cost in power.

更一般地：**每一个一致性协议，都在"物理层面无法维持全序"和"逻辑层面需要某种一致性"之间选择了一个操作点。** 该操作点的选择取决于：(1) 物理延迟的可控程度，(2) 你可以承受的功耗预算，(3) 上层软件能容忍的"不一致性"程度。

More generally: **every coherence protocol chooses an operating point between "physically impossible to maintain total order" and "logically requires some form of consistency."** The choice of operating point depends on: (1) how controllable the physical latency is, (2) the power budget you can afford, (3) how much "inconsistency" the software layer above can tolerate.

---

**虚拟内存**不应被描述为"地址的 JPEG"。JPEG 是有损压缩：原图像与解压图像不相等。虚拟内存恰恰相反：**程序不能察觉自己的虚拟地址被翻译为物理地址。** 这是一种无损的编码转换——映射函数 f: V → P ∪ {⊥}（其中 ⊥ = 缺页）在缺页之外是双射。

**Virtual memory** should not be described as "the JPEG of addresses." JPEG is lossy compression: the original image and the decompressed image are not equal. Virtual memory is the opposite: **a program cannot detect that its virtual addresses are being translated to physical addresses.** This is a lossless encoding transform — the mapping function f: V → P ∪ {⊥} (where ⊥ = page fault) is a bijection outside of page faults.

虚拟内存解决了一个不同的管理问题。每个进程在物理上只能看到一个共享的物理内存池。但每个进程在逻辑上期望独占整个地址空间。这种"多对一"的映射使得：你可以在多个不信任彼此的进程之间共享一台物理机器（进程隔离）；你可以给一个进程分配非连续的物理页，同时让它看到连续的虚拟地址空间（碎片整理）；你可以仅在进程真正需要访问某页时，才将它从磁盘加载到物理内存（按需分页）。

Virtual memory solves a different management problem. Physically, each process can only see a shared pool of physical memory. Logically, each process expects exclusive access to the full address space. This many-to-one mapping enables: running mutually-untrusting processes on one physical machine (process isolation); giving a process non-contiguous physical pages while presenting a contiguous virtual address space (defragmentation); loading a page from disk into physical memory only when the process actually needs to access it (demand paging).

缺页中断（page fault）是局部的断裂——一个物理事实（页不在内存中）打断了逻辑流程（程序认为自己在访问可用的内存地址）。这和第五章相对论节中的观察——测量瞬间，"确定性流动"和"随机断裂"共存——形成了形式上的对偶。一个页错误对程序而言是不可预测的，但其处理流程是完全确定性的。

A page fault is a rupture of locality — a physical fact (the page is not in memory) interrupting the logical flow (the program believes it is accessing an available address). This forms a formal duality with the observation from Chapter 5's relativity section — at the moment of measurement, "deterministic flow" and "random rupture" coexist. A page fault is unpredictable from the program's perspective, yet its handling procedure is completely deterministic.

---

**布隆过滤器**是第四章"随机性作为货币"在空间维度的应用。

**The Bloom filter** is the application of Chapter 4's "randomness as a currency" in the spatial dimension.

一个集合 S 包含 n 个元素。你想支持查询"x 是否在 S 中"。哈希表可以给出确定的答案——但需要 O(n) 空间。布隆过滤器按以下方式给答案：你分配 m 个比特的数组，用 k 个独立的哈希函数将每个元素映射到 k 个比特位上。查询 x 时，检查这 k 个比特是否全为 1。如果全为 1，答案是**"x 可能在 S 中"**。如果有任何一个比特为 0，答案是**"x 绝对不在 S 中"**。

A set S contains n elements. You want to support queries of the form "is x in S?" A hash table can give a deterministic answer — but requires O(n) space. The Bloom filter answers as follows: allocate an m-bit array, and use k independent hash functions to map each element to k bit positions. To query x, check if all k bits are 1. If all are 1, the answer is **"x may be in S."** If any bit is 0, the answer is **"x is definitely not in S."**

布隆过滤器的假阳性率是：

The false positive rate of a Bloom filter is:

$$\varepsilon \approx (1 - e^{-kn/m})^k$$

当 k ≈ (m/n) ln 2 时，ε 取最小值。此时每个元素大约需要 1.44 log₂(1/ε) 比特的空间。例如，ε = 1% 时，每个元素约需 10 比特——远小于在哈希表中存储元素本身的成本。

The minimum occurs at k ≈ (m/n) ln 2. At this optimum, each element requires approximately 1.44 log₂(1/ε) bits of space. For example, at ε = 1%, each element requires roughly 10 bits — far less than the cost of storing the element itself in a hash table.

布隆过滤器和伪随机数生成器共享同一个逻辑结构：**用一个确定性的、可重复的计算过程，制造出统计上的保证——而非确定性的保证。** 你支付"假阳性的微小概率"，换取"空间成本的大幅下降"。这是用确定性换效率——正如第二章的兰道尔原则在物理层面用能量换信息——的又一具体案例。

The Bloom filter and the pseudo-random number generator share the same logical structure: **using a deterministic, reproducible computation to produce statistical guarantees — rather than deterministic ones.** You pay "a small probability of false positives" in exchange for "a massive reduction in space cost." This is another concrete instance of trading determinism for efficiency — just as Landauer's principle trades energy for information at the physical level.

---

局部性，最终是一个关于结构的事实。

Locality, ultimately, is a fact about structure.

一个程序的内存访问流之所以可预测——因此可被层级加速——是因为该访问流具有**结构**。这个结构不依赖于程序员的意图或品味；它来自于算法本身的逻辑约束。一个遍历数组的循环重复地访问连续地址，因为数组的元素在内存中就是连续存放的。一个递归下降解析器反复访问栈顶附近的内存，因为调用图的结构本身就是树状的。结构是客观的，局部性是它的统计足迹。

A program's memory access stream is predictable — and therefore accelerable by hierarchy — because that access stream has **structure**. This structure does not depend on the programmer's intent or taste; it arises from the logical constraints of the algorithm itself. A loop iterating over an array repeatedly accesses contiguous addresses because array elements are laid out contiguously in memory. A recursive descent parser repeatedly accesses memory near the top of the stack because the call graph itself is tree-structured. Structure is objective; locality is its statistical footprint.

第二章定义了"压缩率 = 一个东西有多少结构的客观度量"。局部性的程度——条件熵 H(addr_{t+1} | addr_t) 低于无条件熵 H(addr) 的差距——就是内存访问流中"可压缩结构"的精确度量。而整个存储层级，不过是一台**硬件的解压引擎**——它利用这个结构，将一个小而快的顶层空间自动展开为大而慢的底层空间，而程序几乎注意不到这一点。

Chapter 2 defined "compression ratio is an objective measure of how much structure a thing has." The degree of locality — the gap between the conditional entropy H(addr_{t+1} | addr_t) and the unconditional entropy H(addr) — is the precise measure of "compressible structure" in the memory access stream. And the entire memory hierarchy is nothing more than a **hardware decompression engine** — exploiting this structure to automatically unfold a small, fast top-level space into a large, slow bottom-level space, with the program barely noticing.

---

**局部性是程序的客观性质。层级是利用这个性质的工程手段。前者可测量，后者可设计——但在二者之上，物理极限不可谈判。** 光速给距离的延迟设了下界；兰道尔界限给擦除的能耗设了下界。在这两个下界构成的牢笼中，层级和局部性穷尽了计算机架构师的全部操作空间。

**Locality is an objective property of programs. Hierarchy is the engineering means of exploiting that property. The former can be measured; the latter can be designed — but above both, physical limits are non-negotiable.** The speed of light sets a lower bound on latency across distance; Landauer's bound sets a lower bound on energy for erasure. In the cage formed by these two lower bounds, hierarchy and locality exhaust all the operating space available to the computer architect.
