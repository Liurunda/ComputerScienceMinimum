**练习 1 — 随机性作为计算资源：蒙特卡洛估算 π**

**Exercise 1 — Randomness as a Computational Resource: Monte Carlo Estimation of π**

蒙特卡洛方法用随机撒点来近似确定性的量。在这个经典实验中，你将用随机性来近似 π，并观察随机性作为"计算资源"的效率和局限。

The Monte Carlo method uses random sampling to approximate deterministic quantities. In this classic experiment, you will approximate π using randomness, and observe the efficiency and limits of randomness as a "computational resource."

---

**(a) 基础实现**

**(a) Basic implementation**

在一个边长为 2 的正方形内均匀随机撒点，正方形内嵌一个半径为 1 的圆。撒 N 个点，统计落入圆内的点的比例。这个比例 ≈ π/4。用 Python（或你熟悉的语言）实现。

Uniformly scatter random points inside a 2×2 square that contains an inscribed circle of radius 1. Scatter N points, count the proportion that fall inside the circle. This proportion ≈ π/4. Implement this in Python (or your preferred language).

对于 N = 10², 10³, 10⁴, 10⁵, 10⁶，分别运行 10 次。记录每次的 π 估计值的误差 |estimate − π|。绘制误差的均值（以及标准差）随 N 变化的曲线，标注 1/√N 参考线。你的误差曲线是否大致符合 1/√N 的行为？在什么 N 以上，误差开始小于 0.01？小于 0.001？

For N = 10², 10³, 10⁴, 10⁵, 10⁶, run 10 trials each. Record the error |estimate − π| for each trial. Plot the mean error (and standard deviation) against N, with a 1/√N reference line. Does your error curve roughly follow 1/√N? Above what N does the error drop below 0.01? Below 0.001?

---

**(b) 与确定性方法对比**

**(b) Comparison with a deterministic method**

改用中点法（midpoint rule）数值积分估算 π：将区间 [−1, 1] 均匀划分为 M 个子区间，在每个子区间中点计算 √(1−x²)，累积面积。对于不同的 M 值，记录误差。与 (a) 中蒙特卡洛方法的误差-成本曲线对比。

Switch to the midpoint rule for numerical integration to estimate π: divide [−1, 1] into M equal subintervals, evaluate √(1−x²) at each midpoint, accumulate area. For different M, record the error. Compare the error-versus-cost curve with the Monte Carlo method from (a).

哪个方法在低精度预算（少量函数评估）下表现更好？哪个方法在高精度预算下表现更好？为什么蒙特卡洛方法在 100 维空间中可能优于中点法，而在一维的 π 估算中则未必？

Which method performs better under low precision budget (few function evaluations)? Under high precision budget? Why might Monte Carlo outperform the midpoint rule in 100-dimensional space, yet not necessarily in 1-dimensional π estimation?

**(c) 确定性 seed 的悖论**

**(c) The paradox of deterministic seeding**

修改你的蒙特卡洛程序：在每次运行前显式设置随机数生成器的 seed 为一个固定值（如 42）。现在"随机"实验变得完全可复现——任何人都可以用同一个 seed 精确重现你的结果。

Modify your Monte Carlo program: before each run, explicitly set the random number generator seed to a fixed value (e.g., 42). Now your "random" experiment is fully reproducible — anyone with the same seed can exactly recreate your results.

回答：如果你的蒙特卡洛程序是完全可复现的，那么它还是"随机"算法吗？它和确定性的中点法在哲学上还有什么区别？你愿意信任一个 seed=42 的蒙特卡洛算法来估算一座桥梁的承重——还是要求用确定性的数值积分？为什么？

Answer: If your Monte Carlo program is fully reproducible, is it still a "random" algorithm? What philosophical difference remains between it and the deterministic midpoint rule? Would you trust a seed=42 Monte Carlo algorithm to estimate the load capacity of a bridge — or would you demand deterministic numerical integration? Why?

**练习 2 — 伪随机的"罪"：种子、可复现性与不可区分性**

**Exercise 2 — The "Sin" of Pseudo-randomness: Seeds, Reproducibility, and Indistinguishability**

---

**(a) 种子的力量**

**(a) The power of the seed**

用你选择的编程语言，生成两段"随机"序列：一段使用密码学安全的随机数生成器（如 Python 的 `secrets` 模块或 `/dev/random`），一段使用普通的伪随机数生成器（如 Python 的 `random` 模块），但将种子设为你选择的一个 64 位整数。

Generate two "random" sequences using your chosen programming language: one using a cryptographically secure random number generator (e.g., Python's `secrets` module or `/dev/random`), the other using an ordinary PRNG (e.g., Python's `random` module) with the seed set to a 64-bit integer of your choice.

对于伪随机序列：不记录种子，只记录序列的前 100 个值。一周之后，你还能重建完全相同的序列吗？对于密码学安全的随机序列：同样只记录前 100 个值——你能重建它吗？

For the PRNG sequence: do not record the seed; only record the first 100 values. One week later, can you reconstruct the exact same sequence? For the cryptographically secure sequence: similarly record only the first 100 values — can you reconstruct it?

**(b) 统计检测的边界**

**(b) The limits of statistical detection**

真随机序列和伪随机序列在**统计上**是否可以区分？如果你不知道一段序列的来源，单凭统计测试，你能否判断它是"真随机"还是"伪随机"？

Can true random and pseudo-random sequences be distinguished **statistically**? If you don't know the source of a sequence, can you determine whether it is "truly random" or "pseudo-random" through statistical tests alone?

- 生成一段足够长的伪随机序列（比如 10⁶ 个比特）和一段等长的真随机序列（来自 `/dev/random` 或 random.org）。
- 对两段序列分别运行以下测试：频数测试（0 和 1 的比例是否接近 1:1？）、游程测试（连续相同的比特段长度的分布）、卡方检验。
- 你的测试能区分它们吗？如果能，是什么暴露了伪随机序列？如果不能，为什么密码学安全的 PRNG（如 Mersenne Twister 的后继者）被认为"统计上不可区分"？

Generate a sufficiently long pseudo-random sequence (e.g., 10⁶ bits) and a true random sequence of equal length (from `/dev/random` or random.org).; Run the following tests on both sequences: frequency test (is the ratio of 0s to 1s close to 1:1?), runs test (distribution of consecutive identical bit run lengths), chi-squared test.; Can your tests distinguish them? If so, what exposed the pseudo-random sequence? If not, why are cryptographically secure PRNGs considered "statistically indistinguishable"?

**(c) Randomness = observer relationship**

**(a) Implement a simple fuzzer**

**(c) Formal verification vs. random testing: a debate**

读一读关于 fuzzing 和形式化验证（formal verification）的一个简短介绍（可以让 LLM 帮你总结两者的核心差异）。然后，写一段简短的辩论稿，从两个立场分别论述：
