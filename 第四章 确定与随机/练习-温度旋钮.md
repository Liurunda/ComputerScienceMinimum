**(a) 探索温度光谱**

**(a) Explore the temperature spectrum**

选择一个支持调节温度参数的大语言模型（通过 API 或本地推理）。选取以下三个任务，分别在 T = 0, 0.3, 0.7, 1.0, 1.5 五个温度值下各运行 5 次：

Choose an LLM that supports temperature adjustment (via API or local inference). Select the following three tasks, and run each at five temperature values — T = 0, 0.3, 0.7, 1.0, 1.5 — 5 times each:

- **任务 A（事实准确性）**：列出三个图灵奖得主及其主要贡献。
- **任务 B（创意生成）**：为一本科幻小说构思三个开头段落。
- **任务 C（代码生成）**：编写一个 Python 函数，判断一个字符串是否是回文。

Task A (factual accuracy): List three Turing Award winners and their main contributions.; Task B (creative generation): Write three opening paragraphs for a science fiction novel.; Task C (code generation): Write a Python function that checks whether a string is a palindrome.

记录：任务 A：哪个温度值开始出现事实错误？每个温度下，5 次结果的一致性如何？任务 B：哪个温度范围产生了"最有创意"的结果？哪个温度开始产生无意义的随机文本？任务 C：代码的正确率在各温度下分别是多少？错误的代码是什么样的错误（语法错误？逻辑错误？API 幻觉？）

Record: Task A: At which temperature do factual errors first appear? How consistent are the 5 outputs at each temperature?; Task B: Which temperature range produces the "most creative" results? At which temperature does the output become meaningless random text?; Task C: What is the code correctness rate at each temperature? What kind of errors appear — syntax errors? logic errors? hallucinated APIs?

**(b) 可视化 tradeoff**

**(b) Visualize the tradeoff**

Plot your results as a "creativity vs. accuracy" scatter plot (one point per temperature, x-axis = accuracy on Task A, y-axis = creativity score on Task B). Label each point with its temperature value.

这条曲线是否印证了本章的核心论点——**T 确实是在确定性和随机性之间连续调节的旋钮**？是否存在某个"最优 T"能同时兼顾准确性和创造力？还是必须在两者之间做排他选择？

Does this curve confirm the chapter's core claim — that **T is indeed a knob for continuous tuning between determinism and randomness**? Is there some "optimal T" that simultaneously achieves both accuracy and creativity? Or must one make an exclusive choice between the two?
